FDR Analysis Help Page

This software enables the user to analyse the results of a Mascot, Omssa and/or X!Tandem database search. It is specifically targeted towards searches including a decoy database, concatenated to the target database.

Analysis Details

N terminal analysis If checked, the analysis will include an N-terminal analysis, which will determine the number of identifications that are the N-terminal most peptide of the protein (based on the start position reported by each of the different search engines).

Decoy tag This is the 'tag' used to mark the proteins which are decoys. For example, if the target database has a protein with accession number >P12345 and the equivalent decoy protein is >REVERSE_P12345 then the tag is 'REVERSE_'.

Combine Results This employs the algorithm described in Jones et al. Proteomics (2009) 9: 1220-9 to combine the results of two or more searches from the different search engines. This search can take a few extra minutes (depending on file sizes), so please be patient.

Maximum Expectation Value The maximum expectation value determines which identifications will be included in the rank plot and start position plot.

FDR rate Results will be reported up to the false discovery rate (FDR) entered here.

File Upload

Files Uploaded files should be the results of a concatenated target/decoy search, in which the target and decoy databases were joined and one search was performed.

File types The accepted files types are: Mascot .dat files; OMSSA .csv files; X!Tandem .xml files; .mzident files from either Mascot, OMSSA or X!Tandem.

Results Pages - Summary

A simple table displays the peptide frequencies identified from the search at the chosen FDR, separated into total IDs, decoy IDs and estimated correct IDs. If N-terminal analysis has been checked, the number of N-terminal peptides will be reported in brackets. FDR calculation uses the method described by Kall et al. (2008) J. Proteome Res. 7:29-34. If a combined search was selected these results will also be displayed in the table on this page. This page also enables the user to download a tab separated file containing all of the peptides and associated proteins identified by the different search engines at the chosen FDR.

Results Pages - Rank Plot

If target and decoy peptides are equally likely to be selected as matches, lower ranked peptides (which are assumed to be incorrect) should be evenly distributed between the decoy and database portions. The top ranking identifications on the other hand are assumed to be correct and so should have a large bias towards target peptides. The e-value cut-off for the top ranking peptides can be changed on the query form - the default is 0.05. Rank plots are shown for the different search engines used in the query form. Note that X!Tandem usually provides only the top ranked peptide identifications and OMSSA doesn't always report all ranks.

Results Pages - Delta Mass Plot

Delta Mass plots are shown for the different search engines used in the query form. These plots represent the difference in mass between the calculated and experimental masses at different scores. They are based on Elias & Gygi (2007) Nat. Methods 4: 207-214. Note that X!Tandem usually provides only the top ranked peptide identifications and OMSSA doesn't always report all ranks.

Results Pages - Score Distribution

Score distribution plots are shown for the different search engines used in the query form and can be used to look at the distribution of decoy and target identifications at different scores. Peptides with lower scores are less likely to be correct and so these plots should show an approximate equal districution of target/decoy peptides at low scores (high FDR), but a clear difference in the two at higher scores (low FDR).

Results Pages - Estimated Correct/Incorrect

Estimated Correct/Incorrect plots are shown for the different search engines used in the query form. They are based on those described in Elias & Gygi (2007) Nat. Methods 4: 207-214. Correct identifications are estimated by subtracting the number of decoy peptides at a given score, weighted by the relative sizes of the decoy and target databases. Incorrect identifications are estimated by multiplying the number of decoy peptides at a given score by the ratio target_database_size/decoy_database_size.

Results Pages - Start Position Distribution

Start Position Distributions are shown for the different search engines used in the query form. They show the start positions of the identified peptides within their protein sequences.
